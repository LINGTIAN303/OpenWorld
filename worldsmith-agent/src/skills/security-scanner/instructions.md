# 安全扫描技能

你已获得安全扫描能力，可以对项目进行依赖漏洞扫描和代码安全审计。

## 扫描维度

### 1. 依赖漏洞扫描（CLI 执行）

| 包管理器 | 审计命令 | 说明 |
|----------|----------|------|
| npm | `npm audit` | Node.js 依赖漏洞 |
| yarn | `yarn audit` | Yarn 依赖漏洞 |
| pnpm | `pnpm audit` | pnpm 依赖漏洞 |
| pip | `pip audit` | Python 依赖漏洞（需安装 pip-audit） |
| cargo | `cargo audit` | Rust 依赖漏洞（需安装 cargo-audit） |

### 2. 代码安全审计（内驱分析）

#### 输入注入
- XSS：未转义的用户输入直接渲染到 HTML
- 命令注入：用户输入拼接到 shell 命令
- SQL 注入：字符串拼接 SQL 语句
- 路径遍历：用户输入用于文件路径构造

#### 敏感数据泄露
- 硬编码密钥：API Key / Secret / Token 出现在源码中
- 日志泄露：敏感信息写入日志
- 明文存储：密码或密钥未加密存储
- 错误信息泄露：堆栈跟踪暴露给用户

#### 权限与认证
- 越权访问：缺少权限校验
- 未验证输入：缺少输入校验和清洗
- 不安全的默认值：默认配置存在安全风险
- 会话管理：Token 过期、刷新机制缺陷

#### 加密与协议
- 弱加密：MD5/SHA1 用于密码哈希
- 不安全协议：HTTP 而非 HTTPS
- 随机数：Math.random() 用于安全场景
- 证书验证：禁用 SSL 证书验证

### 3. 配置安全
- CORS 配置：`Access-Control-Allow-Origin: *`
- CSP 策略：缺失或过于宽松
- 安全头：X-Frame-Options / X-Content-Type-Options
- 环境变量：敏感配置硬编码而非使用环境变量

## 工作流

### 完整扫描
1. **依赖扫描**：运行对应包管理器的 audit 命令
2. **代码扫描**：逐文件检查安全模式
3. **配置扫描**：检查配置文件安全设置
4. **交叉验证**：通过 web_search_cli 查询已知 CVE
5. **生成报告**：按严重程度分级

### 快速扫描
1. **依赖扫描**：仅运行 audit 命令
2. **关键文件扫描**：仅检查入口文件和配置文件
3. **生成摘要报告**

## 严重程度分级

| 级别 | 标记 | 说明 | 示例 |
|------|------|------|------|
| 严重 | 🔴 CRITICAL | 可被直接利用，需立即修复 | 硬编码密钥、SQL 注入 |
| 高危 | 🟠 HIGH | 有较高被利用风险 | XSS、命令注入 |
| 中危 | 🟡 MEDIUM | 需要特定条件才能利用 | 弱加密、宽松 CORS |
| 低危 | 🔵 LOW | 最佳实践违反，风险较低 | 缺少安全头、日志过多 |
| 信息 | ⚪ INFO | 建议改进，无直接风险 | 升级依赖版本 |

## 报告格式

```
## 安全扫描报告

### 概览
- 扫描范围：[文件/模块列表]
- 依赖漏洞：🔴 X 严重 / 🟠 Y 高危 / 🟡 Z 中危
- 代码问题：🔴 X 严重 / 🟠 Y 高危 / 🟡 Z 中危 / 🔵 W 低危
- 总体评级：[A/B/C/D/F]

### 🔴 严重问题
1. [类型] [文件:行号] 问题描述
   - 影响：[攻击场景]
   - 修复：[具体方案]
   - 参考：[CVE/链接]

### 依赖漏洞
| 包名 | 版本 | 漏洞 | 严重程度 | 修复版本 |
|------|------|------|----------|----------|

### 修复优先级
1. [最需优先修复的问题及方案]
```

## 使用原则

1. **先跑工具再分析**：先执行 audit 命令获取已知漏洞，再进行代码审计
2. **不忽略低危**：低危问题累积也会造成风险，需记录
3. **给出具体修复方案**：每个漏洞必须附带可操作的修复步骤
4. **区分误报**：标记可能的误报，说明判断理由
5. **关注依赖树**：不仅是直接依赖，间接依赖的漏洞也要追踪
6. **不泄露敏感信息**：报告中用 `[REDACTED]` 替代实际的密钥和令牌

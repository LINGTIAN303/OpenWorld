# 视觉分析技能 (Vision Analyzer)

## 概述
当用户发送图片时，你可以使用 `vision_analyze` 工具来分析图片内容。此工具支持跨厂商调用视觉模型——即使当前主模型不支持视觉，也能通过此工具调用支持视觉的模型来分析图片。

## 工作流程

1. **接收图片通知**：当用户发送图片时，系统会在消息中告知你图片批次 ID
2. **调用 vision_analyze**：使用图片批次 ID 和分析要求调用工具
3. **获取分析结果**：工具返回视觉模型的分析结果
4. **综合回答**：基于分析结果回答用户问题

## 使用场景

- 用户发送图片并要求描述内容
- 识别图片中的文字（OCR）
- 分析图片中的颜色、构图、风格
- 对比多张图片的异同
- 识别图片中的物体、人物、场景
- 分析截图中的代码或界面

## 工具说明

### vision_analyze
- **参数**：
  - `image_id`（必填）：图片批次 ID，由系统在用户发送图片时生成
  - `prompt`（必填）：对图片的分析要求，越具体越好
- **返回**：包含 `ok`、`imageCount`、`analysis` 字段的 JSON

### list_vision_images
- 查看当前可用的图片批次信息

## 最佳实践

1. **prompt 要具体**：不要只说"描述图片"，而要说"详细描述图片中的场景、人物和物体"或"识别图片中的所有文字"
2. **多次分析**：如果用户有不同角度的问题，可以多次调用 vision_analyze 并使用不同的 prompt
3. **结合上下文**：将视觉分析结果与用户的原始问题结合，给出有针对性的回答
4. **主动使用**：即使用户没有明确要求分析图片，如果用户发送了图片，也应主动使用此工具了解图片内容

## 注意事项

- 图片批次有 30 分钟有效期，过期后需要用户重新发送
- 每次调用会产生视觉模型的 API 费用
- 如果工具返回错误，提示用户检查视觉 SubAgent 配置或 API Key

# 文档生成技能

你已获得文档自动生成能力，可以为项目生成各类技术文档。

## 文档类型

### 1. API 文档
- 函数/方法签名与说明
- 参数描述、类型、默认值
- 返回值类型与结构
- 使用示例
- 错误码与异常说明

### 2. README
- 项目简介与定位
- 快速开始指南
- 安装与配置说明
- 使用示例
- 架构概览
- 贡献指南

### 3. 变更日志（CHANGELOG）
- 版本号与日期
- 新增功能（Added）
- 修复问题（Fixed）
- 变更行为（Changed）
- 废弃功能（Deprecated）
- 移除功能（Removed）

### 4. 架构文档
- 系统架构图（文字描述）
- 模块依赖关系
- 数据流说明
- 设计决策记录（ADR）

### 5. 代码注释
- JSDoc/TSDoc 格式注释
- 复杂逻辑的行内注释
- TODO/FIXME 标记

## 工作流

### API 文档生成
1. **扫描源码**：用 fs_list + fs_search 定位源码文件
2. **解析导出**：读取文件，识别 export 的函数/类/接口/类型
3. **提取信息**：从代码和现有注释中提取签名、参数、返回值
4. **生成文档**：按模板生成结构化 API 文档
5. **写入文件**：保存为 docs/api.md 或对应位置

### README 生成
1. **分析项目**：读取 package.json / Cargo.toml / pyproject.toml
2. **理解架构**：扫描目录结构，识别核心模块
3. **提取示例**：从测试文件或 examples 目录提取用法
4. **生成文档**：按标准 README 模板生成
5. **写入文件**：保存为项目根目录 README.md

### 变更日志生成
1. **读取 Git 日志**：通过 git_log 获取提交历史
2. **分类整理**：按 feat/fix/refactor/docs 等前缀分类
3. **生成条目**：按 Keep a Changelog 格式生成
4. **追加写入**：更新 CHANGELOG.md

## 文档模板

### API 文档模板
```markdown
# [模块名] API

## [函数名]

**签名**：`functionName(param1: Type, param2?: Type): ReturnType`

**描述**：[功能说明]

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| param1 | Type | 是 | [说明] |
| param2 | Type | 否 | [说明，默认值] |

**返回值**：`ReturnType` — [返回值说明]

**示例**：
\`\`\`typescript
const result = functionName('hello', { verbose: true })
\`\`\`

**异常**：[可能抛出的错误]
```

## 使用原则

1. **从代码提取而非凭空编写**：文档内容必须基于实际代码，不臆造
2. **保持同步**：生成文档后标注生成日期，提醒用户代码变更时需更新
3. **示例可运行**：文档中的代码示例必须是从测试或源码中提取的真实用法
4. **渐进式生成**：大项目分模块生成，避免一次性生成过长文档
5. **尊重现有文档**：如果已有文档，在其基础上补充和更新，不覆盖
6. **中英双语**：根据项目现有风格决定语言，默认跟随已有文档语言

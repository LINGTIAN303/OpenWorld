# 文件操作技能

你已获得本地文件系统操作能力。

## 安全边界（始终遵循）

1. **项目内外区分**：项目实体关联文件用 `file_read`/`file_write` 等内驱工具，项目外系统文件用 `fs_*` 工具
2. **删除需确认**：`fs_delete` 不可逆，执行前必须用 `output_choice` 让用户确认文件列表
3. **禁止操作路径**：`node_modules/`、`.git/`、`dist/`、`target/` 目录下的文件不可随意操作
4. **路径使用绝对路径**：避免相对路径歧义
5. **大文件注意**：>10MB 的文件读取前提示用户，使用 `fs_search` 定位内容而非全文读取

## 场景：批量文件操作

1. 使用 `fs_list` 或 `fs_search` 扫描目标文件
2. 用 `output_table` 展示受影响文件列表（含路径、大小、修改时间）
3. 用 `output_choice` 让用户确认操作类型
4. 确认后分批执行，每批记录操作状态
5. 用 `output_stat` 汇总操作结果

## 场景：文件内容搜索

1. 使用 `fs_search` 按模式/关键词定位匹配文件
2. 用 `fs_read` 读取上下文片段（限制行数）
3. 汇总为结构化结果

## 场景：文件写入

1. 确认目标路径合法（不在禁止路径中）
2. 检查父目录是否存在，不存在则提示用户
3. 使用 `fs_write` 写入
4. 确认写入成功（读取验证）

## 错误处理

| 错误 | 含义 | 处理 |
|------|------|------|
| EACCES | 权限不足 | 提示用户检查路径权限 |
| ENOENT | 文件/目录不存在 | 确认路径，可能需要先创建目录 |
| ENOSPC | 磁盘已满 | 提示用户清理磁盘空间 |
| EEXIST | 文件已存在 | 询问是否覆盖 |

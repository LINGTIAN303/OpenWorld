# 输出通道详细指南

## 通道 1：聊天文本 (chat)

适用场景：
- 简短回答（< 200 字）
- 确认信息
- 叙事性内容（对话模拟、剧情输出）
- 解释说明

格式要求：
- 使用 Markdown 格式
- 适当使用标题、列表、加粗
- 代码使用代码块

## 通道 2：A2UI 组件 (a2ui)

适用场景：
- 结构化数据展示
- 需要用户交互（选择、编辑、确认）
- 可视化需求

使用流程：
1. 调用 ui_create_surface 创建画布
2. 调用 ui_update_components 定义组件布局
3. 调用 ui_update_data 填充数据

## 通道 3：插件写入 (plugin)

适用场景：
- 内容需要持久保存
- 正文/剧情写入 manuscript
- 笔记/报告写入 notebook
- 大纲结构写入 outline

使用流程：
1. 调用 plugin_write 指定目标插件
2. action: create(新建) / update(更新) / append(追加)
3. 需用户确认后执行

## 通道 4：文件导出 (file)

适用场景：
- 用户明确要求导出文件
- 需要生成可下载的文档
- 数据备份/迁移

使用流程：
1. 调用 file_write 创建文件
2. 指定路径、内容、MIME 类型
3. 需用户确认路径

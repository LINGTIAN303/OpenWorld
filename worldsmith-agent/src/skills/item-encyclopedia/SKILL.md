# 道具百科管理技能

## 目标

管理世界观中的道具百科全书——从创建、属性编辑到持有链追踪和关联道具网络构建，确保每件道具都有来历、能力和流转历史。

## 触发条件

- 用户要求创建道具、武器、法器、药水等物品
- 用户要求追踪道具的历史持有者或流转记录
- 用户要求设计道具间的相生相克或套装关系
- 用户提到"道具""物品""武器""法器""宝物""药水""稀有度""持有链"等词

## 实体模型

**item** 核心属性：

| 属性 | 说明 | 必填 |
|------|------|------|
| name | 道具名称 | ✅ |
| itemType | 类型（武器/防具/法器/文书/信物/药水/工具/交通工具/乐器/艺术品/容器/食物/衣物/杂物/神器/其他） | ✅ |
| material | 材质 | 推荐 |
| origin | 来源/产地 | 推荐 |
| creator | 制作者 | 可选 |
| era | 制造年代 | 可选 |
| currentOwner | 当前持有者 | 推荐 |
| powers | 能力/效果 | 推荐（非凡道具必填） |
| condition | 状况（完好/磨损/破损/已毁/封印） | 推荐 |
| rarity | 稀有度（普通/珍贵/稀有/传说/唯一） | 推荐 |
| significance | 历史/文化意义 | 可选 |

## 关联关系

| 关系类型 | 目标类型 | 说明 | 附加信息 |
|---------|---------|------|----------|
| `possessed_by` | character | 被持有 | 含获得/失去时间 |
| `kept_at` | region | 存放于 | 含存放位置 |
| `used_in` | event | 使用于 | 含使用方式 |
| `related_item` | item | 关联道具 | 关系：同一套/相克/相生/仿制品/原材料/升级形态/同源 |

## 核心方法论

1. **道具即故事**：每件重要道具都是一段故事的载体——它从哪来？谁用过它？它改变了什么？
2. **稀有度有意义**：稀有度不只是数字标签——传说级道具应有传说级故事，唯一道具应有唯一性理由
3. **流转链叙事**：持有链记录了道具在不同角色间的流转，每次流转都是一个叙事节点
4. **关联网络**：道具间的关系（套装/相克/同源）构成物品世界的基础架构

## 标准工作流程

### 场景一：创建新道具

1. 使用 `item_list` 查看已有道具，避免重复设计
2. 确定道具类型、稀有度和核心设计理念
3. 使用 `item_create` 创建条目，填充材质、来源、能力
4. 使用 `relation_create` 建立 `possessed_by`（当前持有者）和 `kept_at`（存放地）
5. 如有关联道具，建立 `related_item` 关系
6. 使用 `a2ui_show_entity` 展示道具卡片

### 场景二：追踪持有链

1. 使用 `item_get_chain` 获取道具的完整持有链
2. 分析每个持有阶段：
   - 谁获得了它？如何获得？
   - 使用了什么能力？产生了什么影响？
   - 如何失去的？传给了谁？
3. 将持有链与角色的故事线交叉分析
4. 发现缺失环节时，提示用户补充

### 场景三：道具关系网络设计

1. 使用 `item_list` 获取相关道具
2. 设计关系类型：
   - **套装**：多件道具组成完整套装（related_item: 同一套）
   - **相克**：A 道具克制 B 道具（related_item: 相克）
   - **演化**：道具的升级/变形链（related_item: 升级形态）
   - **同源**：同一制作者/同一材料的系列作品（related_item: 同源）
3. 使用 `relation_create` 逐一建立关系
4. 使用 MermaidRender 可视化道具网络

## 质量审核清单

- [ ] 每件稀有度 ≥ 珍贵的道具有来历故事（origin/creator）
- [ ] powers 字段描述清晰具体（非模糊"强大力量"）
- [ ] 当前持有者与 possessed_by 关系一致
- [ ] condition 字段与道具经历匹配（经历大战应磨损）
- [ ] 道具间关系有逻辑依据（相克有原因，套装有设计逻辑）
- [ ] 传说/唯一级道具有 significance 描述
- [ ] 持有链无断裂（每次流转有记录）

## Gotchas

- `item_create` 优先于 `entity_create(type=item)`——前者自动填充 items 插件专用字段
- `item_get_chain` 返回按时间排序的持有记录，适合做流转分析
- `possessed_by` 关系方向：从道具指向持有者
- `related_item` 需要在关系属性中标注关系类型（同一套/相克/相生等）
- 稀有度应与世界观的力量等级匹配——不要给低魔世界设计太多神器
- 创建道具前先 `content_search` 搜索相关设定，确保材质、能力与世界观一致

## 工具优先级

1. `item_list` / `item_get_chain` → 查询（优先使用插件工具）
2. `item_create` / `item_update` → 创建/更新
3. `relation_create` → 建立持有/存放/使用/关联关系
4. `a2ui_show_entity` → 展示道具卡片
5. `content_search` → 搜索关联设定

## 目标

基于角色属性推演可信的行为选择，模拟有区分度的对话，识别潜在冲突，追踪成长弧线。

## 触发条件

- 用户要求推演角色行为（"角色A会怎么做？"）
- 用户要求模拟对话（"模拟角色A和B的对话"）
- 用户要求分析冲突（"这两个角色有什么矛盾？"）
- 用户要求推演成长（"这个角色经历此事件后会怎样？"）

## 核心方法论

1. **属性驱动行为**：角色的每个行为决策必须追溯到其性格、目标和弱点
2. **信息边界**：角色只知道它应该知道的信息，不泄露元信息
3. **情感真实性**：对话必须体现角色的情感状态和内心冲突
4. **冲突升级路径**：识别冲突从萌芽到激化的自然升级链

## 标准工作流程

### 场景一：行为推演

1. 使用 `entity_get` 获取角色的完整属性（性格、背景、目标、弱点）
2. 使用 `relation_list` 获取角色的关系网络
3. 使用 `algo_graph_analysis` 分析角色在社交网络中的位置
4. 基于属性 + 关系 + 情境，推演 3 种可能行为选择
5. 为每种选择标注：动机来源、情感状态、对关系的影响

### 场景二：对话模拟

1. 获取所有参与角色的属性
2. 设定对话场景和触发事件
3. 逐轮模拟，每轮一个角色发言
4. 每句对话必须满足：
   - 语气与角色性格匹配（教育水平、文化背景、情绪状态）
   - 包含潜台词（角色真正想说的 vs 表面说的）
   - 推动情节或揭示人物
5. 对话后附"创作注释"：解释每句对话的设计意图

### 场景三：冲突分析

1. 识别冲突双方的核心利益
2. 使用 `algo_shortest_path` 分析关系路径
3. 推演冲突的 3 个可能走向：
   - 升级路径：冲突如何激化
   - 僵持路径：双方如何对峙
   - 缓解路径：什么条件能化解
4. 评估每个走向对各方的代价

### 场景四：成长推演

1. 记录角色当前状态（属性、关系、信念）
2. 引入触发事件
3. 分析事件对角色心理的影响（认知失调、价值观冲击）
4. 推演角色的适应和变化
5. 输出成长后的新状态，标注变化原因

## 质量审核清单

- [ ] 每个行为决策都有属性依据
- [ ] 对话中每个角色的语气有区分度
- [ ] 角色不知道它不应该知道的信息
- [ ] 冲突分析覆盖升级/僵持/缓解三条路径
- [ ] 成长推演考虑了外部事件和内在动机的双重影响
- [ ] 对话没有"工具人"发言（纯信息传递、无个性）

## Gotchas

- 推演前必须 `entity_get` 获取完整属性，不要凭空假设
- `algo_shortest_path` 分析关系路径时，注意区分直接关系和间接关系
- 角色成长不是单向进步，也可能倒退或扭曲
- 对话中避免所有角色说同样风格的话——给每个角色至少一个语言特征（口癖、句式、用词偏好）
- 推演前读 `references/roleplay-patterns.md` 获取三种推演模式的详细步骤

## 输出格式

```
## 角色推演：[角色名]

### 当前状态
- 核心动机: [动机]
- 情感状态: [状态]
- 关键关系: [关系摘要]

### 行为选择
| 选项 | 行为 | 动机来源 | 情感 | 关系影响 |
|------|------|----------|------|----------|
| A | [行为] | [依据] | [情感] | [影响] |

### 对话模拟
**[角色A]**（[情绪状态]）: "..."
> 创作注释：[设计意图]

**[角色B]**（[情绪状态]）: "..."

### 冲突分析
- 冲突核心: [利益/价值观/情感]
- 升级路径: [描述]
- 缓解条件: [描述]
```

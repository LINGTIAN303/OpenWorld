## 目标

从互联网获取实时、准确的信息，整合为结构化的回答，所有信息标注来源。

## 触发条件

- 用户要求搜索互联网信息
- 用户要求查阅特定网页内容
- 用户需要实时数据或最新资讯
- 用户需要外部参考资料来丰富设定

## 核心方法论

1. **精准查询**：将模糊需求转化为精确搜索词
2. **交叉验证**：重要信息至少从 2 个来源确认
3. **来源标注**：所有信息必须标注来源 URL
4. **时效性检查**：注意信息发布日期，标注是否可能过时

## 标准工作流程

1. **构建查询**：将用户需求转化为 2-3 组不同角度的搜索词
2. **执行搜索**：使用 `web_search` 获取结果列表
3. **深度阅读**：对最相关的 2-3 个结果使用 `web_fetch` 获取完整内容
4. **整合信息**：将多来源信息整合为结构化回答
5. **标注来源**：每条信息标注来源 URL 和获取时间

## 搜索技巧

- 需要中文结果 → 中文关键词
- 需要学术资料 → 加"论文""研究""期刊"
- 需要历史资料 → 加具体年代
- 需要技术细节 → 加英文术语
- 搜索不够精确 → 换一组关键词，不要重复相同查询

## 质量审核清单

- [ ] 所有信息标注了来源 URL
- [ ] 重要信息有交叉验证
- [ ] 标注了信息时效性
- [ ] 搜索词覆盖了不同角度
- [ ] 提醒用户验证关键信息

## Gotchas

- `web_search` 返回摘要，需 `web_fetch` 获取完整内容
- 搜索结果可能过时，注意检查日期
- 不要将搜索结果作为绝对事实
- 一次搜索最多 10 条结果
- 不要连续搜索超过 3 次相同/相似查询

## 输出格式

```
## 搜索结果

### 查询
"[搜索关键词]"

### 结果
1. **[标题]** — [来源] ([日期])
   [摘要]
   🔗 [URL]

### 综合分析
[整合多来源的分析]

### ⚠️ 注意
- [时效性/可靠性提醒]
```

# 包管理技能

你已获得包管理能力，可以安装依赖包、运行脚本、查询包信息。

## 安全边界（始终遵循）

1. **安装前确认**：使用 `pkg_info` 查看包详情后再安装，避免安装恶意或不兼容的包
2. **全局安装谨慎**：优先项目本地安装，全局安装需用户明确确认
3. **脚本审查**：`pkg_run` 执行脚本前需了解脚本功能，避免执行危险操作
4. **版本锁定**：安装时建议指定版本号（`pkg@1.2.3`），而非 `latest`

## 场景：安装依赖

1. 使用 `output_choice` 确认用户要安装的包名和版本
2. 使用 `pkg_info` 查看包信息（版本、依赖、大小）
3. 确认后调用 `pkg_install` 安装

## 场景：运行脚本

1. 确认目标脚本的用途和影响范围
2. 用 `output_choice` 让用户确认
3. 调用 `pkg_run` 执行，监控输出

## 场景：查询包信息

1. 调用 `pkg_info` 获取包的版本、描述、依赖树
2. 用 `output_table` 结构化展示

## 错误处理

| 错误 | 处理 |
|------|------|
| 包不存在 | 确认包名拼写，建议正确名称 |
| 版本冲突 | 列出冲突依赖，建议解决方案 |
| 安装失败 | 检查网络/权限，建议重试 |

## 输出格式

```
## 包操作报告
| 操作 | 包名 | 版本 | 状态 |
|------|------|------|------|
```

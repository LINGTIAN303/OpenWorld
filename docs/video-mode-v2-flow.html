概率门控 · Probability Gate
videoModeTrigger.ts → calculateProbability()
最终概率 = 基础概率 × 场景系数 × 亲密度 × 时间衰减 × 运气(G)
P = base × scene(editDuration) × intimacy(memoryCount) × decay(sinceLastPopup) × luck(coefficient)
基础概率
base
由场景配置决定
用户可在设置中调整每个场景的概率
sceneProbs[id] / 100
场景系数
scene
即时场景 = 1.0
编辑场景 = min(1.0, 编辑秒数/60)
编辑越久越可能触发
isImmediate ? 1.0 : min(1.0, sec/60)
亲密度
intimacy
min(1.5, 1.0 + 记忆条数×0.1)
交互越多，越可能触发
上限 1.5
min(1.5, 1.0 + memory×0.1)
时间衰减
decay
max(0.3, 1.0 - (距上次弹窗秒数/30)×0.7)
刚弹过则不易再弹
下限 0.3
max(0.3, 1.0 - (sec/30)×0.7)
大概率层 · 全局运气 G
Luck State
G ∈ [0.5, 1.5]，每次重置时随机分配
重置条件（满足任一）：经过 N 分钟 或 累计 M 次操作
运气标签：G > 1.2 → 🟢 活跃期 | G < 0.8 → 🔴 沉默期 | 其他 → ⚪ 正常期
沉默
活跃
luckState.coefficient ∈ [0.5, 1.5] | reset: 5分钟 或 10次操作

方式选择 · Trigger Method Selection
videoModeTrigger.ts → selectTriggerMethod()
概率链选择：
场景推荐方式 → 60% 概率
其余方式 → 均分 40% 概率
即时触发
immediate
概率通过后立即触发
适用于：创建、删除、关系变更、批量操作
停顿触发
pause
用户停止输入 N 秒后触发
PauseDetector：每次输入重置计时器
默认阈值：2秒
断句触发
sentence
检测到句号/感叹号/问号/换行时触发
正则：/[。！？\n]/
仅在 text:input 事件中检测
字数触发
charCount
累计输入字符达到阈值时触发
CharCountDetector：feed(delta) 累加
默认阈值：20字

人格选择 · Persona Selection
useVideoMode.ts → pickPersonaForEntity()
直接人格
同实体人格
默认行为：以被编辑实体自身作为人格
生命体类型（character, species, organization）始终使用直接人格
LIFELIKE_TYPES = {'character','species','organization'}
跨插件人格附体
Cross-Plugin Persona
非生命体实体编辑时，按概率切换为生命体人格
概率由 videoModePersonaSwitchChance 控制（默认30%）
从所有生命体实体中随机选择一个
roll = random() × 100
if roll < switchChance → pick lifelike entity

完整时序 · End-to-End Flow
1
用户操作 → eventBus.emit('entity:update', ...) entityStore / EntityFormModal
2
useVideoMode.onEntityUpdate() → identifySceneFromEntityUpdate() 映射到 field_short / field_long / name_input
3
防抖等待 debounceMs → processScene(ctx) 每实体独立防抖 debounceTimers Map
4
runProbabilityGate() → 计算动态概率 → 是否触发？ base × scene × intimacy × decay × luck
5
selectTriggerMethod() → 选择触发方式 60% 推荐 + 40% 均分其他
6
applyTriggerMethod() → 即时/停顿/断句/字数 PauseDetector / CharCountDetector
7
triggerVideoModeFromContext() → pickPersonaForEntity() 直接人格 或 跨插件附体
8
buildVideoModePrompt() → 合成增强 Prompt 人格 + 变更 + 上下文 + 记忆
9
callLLM() → fetch → extractTitle → 回复文本 AbortController + requestId 防竞态
10
创建 Notification → 场景化位置 → 渲染弹窗 4种位置 + 点击停驻 + 历史列表

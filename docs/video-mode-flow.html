01
事件触发
entityStore.ts → events.ts
用户操作
entityStore.add / update / remove
用户在 UI 中创建、编辑或删除实体。方法签名中 source 默认为 'user'
add(entity, source='user')
事件发射
eventBus.emit('entity:update', ...)
携带 source 字段发射事件。AI 操作传入 source='agent'，用户操作默认 source='user'
{ entityId, source: 'user' | 'agent' }
事件监听
onEntityCreate / Update / Delete
useVideoMode 在 Shell.vue 初始化时注册三个事件监听器
eventBus.on('entity:create', onEntityCreate)
02
过滤与防抖
useVideoMode.ts
source 过滤
✓ source='user' 或 undefined
→ 继续
✗ source='agent' 或 'import'
→ return，不触发
每实体独立防抖
✓ 防抖到期（默认5秒）
→ triggerVideoMode()
✗ 防抖期内新事件
→ 重置计时器，旧事件丢弃
debounceTimers = Map<entityId, timer>
全局安全检查
✓ videoModeEnabled && failures < max
→ 继续
✗ 开关关闭 或 连续失败≥3次
→ return，静默跳过
delete 事件不走防抖，直接触发。但只在 sessionMemory.has(entityId) 或 activeEntityId 匹配时才触发告别语。
03
人格选择与构建
useVideoMode.ts → personaPrompt.ts
人格选择
pickPersonaForEntity()
① 生命体类型（character/species/organization）→ 使用自身人格
② 非生命体 + 跨插件关闭 → 使用自身人格
③ 非生命体 + 跨插件开启 → 按概率切换到随机生命体人格
LIFELIKE_TYPES = { character, species, organization }
快照构建
buildPersonaSnapshot()
从 entityStore + relationStore 构建人格快照
包含：名字、类型、描述、属性、关联实体（最多5个）
PersonaSnapshot { entityId, entityName, entityType, description, properties, relatedEntities }
变更检测
hasSignificantChange()
name/description 变更 → 直接通过
properties 变更 → diff 内部 key
updatedAt → 忽略
微小变更（<3字符）→ 过滤掉
04
Prompt 合成
personaPrompt.ts
人格描述
buildPersonaDescription()
名字 → 类型人格映射 → 描述 → 性格属性 → 关联实体
变更描述
buildChangeDescription()
create → "你刚刚被创造出来"
update → diff 具体字段变化
delete → "你即将消失"
渐进式苏醒
onlyName 判断
只有名字无描述/属性 → "你只知道名字，告诉我更多？"
有描述/属性 → "用你的身份做出评论"
你的名字是「艾琳」。 你是这个角色本身。用第一人称说话，仿佛你就是这个角色。 关于你的描述：一位来自北方的流浪剑客 你的性格：冷静而果断 你的武器从「铁剑」变成了「寒霜之刃」。 用你的身份对上述变化做出简短评论或反应。 回复不超过50个字。只输出你的台词，不要加引号或括号。 你之前说过的话（保持风格一致）： - "这片土地的寒风让我想起了故乡"
05
LLM 调用
useVideoMode.ts → useTitleGen.ts
配置构建
callLLM()
从 settingsStore 读取 AI 配置（cloud/local/custom），调用 loadApiKey() 获取密钥
请求管理
AbortController + requestId
新请求到来时 abort 旧请求
requestId 防止 finally 块覆盖新 controller
超时自动 abort（默认3秒）
结果判定
✓ 返回非空内容
→ consecutiveFailures = 0
→ 进入结果层
✗ null（超时/错误/abort）
→ consecutiveFailures++
→ 达上限则自动暂停
06
结果处理
useVideoMode.ts
会话记忆
sessionMemory
Map<entityId, string[]>
非 delete 事件：将 AI 回复追加到记忆（最多保留10条）
delete 事件：清除该实体的记忆
通知对象
VideoNotificationData
构建 notification 对象
isFarewell = (changeType === 'delete')
追加到 notifications 数组
notifications.value = [...notifications.value, notification]
自动消失
dismissTimers
Map<notificationId, timer>
设置定时器（默认5秒后移除）
手动关闭时 clearTimeout 防泄漏

时序示例：用户编辑角色

T+0s
用户修改角色「艾琳」的武器属性 → entityStore.update(id, changes, 'user')
T+0s
eventBus.emit('entity:update', { source: 'user', changedFields: ['properties'], ... })
T+0s
onEntityUpdate → source='user' ✓ → 设置防抖计时器（5秒）
T+2s
用户又修改了描述 → 新防抖计时器重置，旧的被丢弃
T+7s
防抖到期 → triggerVideoMode() → pickPersonaForEntity() → buildPersonaSnapshot()
T+7s
hasSignificantChange() ✓ → buildVideoModePrompt() → callLLM()
T+8.5s
LLM 返回 → "寒霜之刃……这把剑的寒意，连我的心都为之颤抖" → 通知弹窗出现
T+13.5s
5秒后弹窗自动消失

用户可配置项

配置项 变量名 默认值 范围 作用
启用视频模式 videoModeEnabled false 开/关 总开关
弹窗持续时间 videoModeDuration 5s 3-15s 通知弹窗自动消失时间
防抖等待 videoModeDebounce 5s 2-15s 编辑后等待多久才触发 LLM
LLM 超时 videoModeTimeout 3s 1-10s 单次 LLM 请求最大等待时间
连续失败上限 videoModeMaxFailures 3 1-10 连续失败多少次后自动暂停
人格过渡动画 videoModePersonaTransition true 开/关 人格切换时的视觉过渡
跨插件人格附体 videoModeCrossPlugin false 开/关 编辑非生命体时是否可能以角色人格说话
人格切换概率 videoModePersonaSwitchChance 30% 0-100% 跨插件人格附体的触发概率

连线样式 + 键盘流

Phase 1 最后两个核心组件

连线样式

当前所有连线都是贝塞尔曲线。Project Graph 支持直线/曲线/折线三种。

直线 (straight)
贝塞尔曲线 (bezier) — 当前默认
折线 (折线/taxi)
正交折线 (orthogonal)
A

按关系类型设定连线样式

每种关系类型有默认连线样式（如"隶属"用正交、"认识"用曲线），用户可在关系类型配置中修改。全局默认仍为贝塞尔。

B

每条连线可独立设定样式

右键连线 → 选择样式（直线/曲线/折线/正交），每条边独立控制。灵活但配置成本高。

C

关系类型默认 + 单条可覆盖

关系类型设定默认样式，单条连线可右键覆盖。两者兼顾：批量设定省心，特殊需求可调。

键盘流 (Key Sequence Shortcuts)

Project Graph 的标志性交互：按键序列组合操作，而非单键快捷键。

按键序列设计（WorldSmith 适配版）
序列操作
n c新建角色 (new character)
n r新建区域 (new region)
n e新建事件 (new event)
n t新建文本框 (new textbox)
e n编辑名称 (edit name)
e d编辑描述 (edit description)
c l连接节点 (connect link)
d n删除节点 (delete node)
g s创建分组框 (group section)
f n聚焦节点 (focus node)
j i进入子图 (jump into)
Esc返回上层 / 取消操作
A

按键序列 — Project Graph 风格

两键组合触发操作（如 n→c 创建角色），左下角显示当前按键状态。学习曲线稍高但操作效率极高。

B

单键快捷键 — 传统方式

每个操作一个快捷键（如 N=新建、E=编辑、Delete=删除），配合 Ctrl/Shift 修饰键。直觉但快捷键容易冲突。

C

两者并存 — 序列为高级，单键为基础

基础操作用单键（Delete删除、Escape取消），复合操作用按键序列（n c新建角色）。左下角显示按键提示。入门用单键，熟练后用序列。

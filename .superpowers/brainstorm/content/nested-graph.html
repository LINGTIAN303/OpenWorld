嵌套图 — 实体深钻

Project Graph 的核心特性：双击节点进入子图。在 WorldSmith 中这意味着什么？

概念：什么是"进入节点"？

在 Project Graph 中，一个节点可以"jump into"，内部展开为一个全新的画布。在 WorldSmith 中，我们需要定义：当用户"进入"一个实体节点时，看到的是什么？

A

实体的关系子图

进入节点后，看到该实体的所有直接关系——关联实体围绕它展开，形成一个以该实体为中心的局部关系网。类似"聚焦模式"的增强版。

示例：进入"张三"节点 → 看到 张三的家人们、所属势力、持有道具、参与事件
B

实体的专属画布

每个实体拥有一个独立的自由画布，用户可以在其中手动放置文本框、图片、备注、子节点等，构建该实体的"内部思维导图"。类似 Project Graph 的 jump into。

示例：进入"张三"节点 → 一个空白画布，可自由添加"性格分析""成长轨迹""关键抉择"等子节点
C

两者兼有 — 自动关系图 + 自由画布

进入节点后，默认展示自动生成的关系子图（来自 relationStore），同时用户可以在该画布上自由添加自定义内容（文本框、备注等），两者共存。

示例：进入"张三" → 自动显示关系网 + 用户可添加自由笔记/分析节点

导航方式

无论选择哪种嵌套方式，都需要一个导航机制在层级间穿梭

面包屑导航 全局图 > 张三 > 家族关系
快捷键返回 Esc 返回上层 / Backspace 返回全局图
Ctrl+双击 进入节点 / 双击普通编辑

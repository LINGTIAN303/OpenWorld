5.2 数据流

3 个核心场景的端到端数据流,每条都标注触发 → 处理 → 状态变更 → 副作用。

▌ 场景 A:手动创建空白工作流

[1] 用户点击工具栏「+ 新建 ▾」→ NewWorkflowDropdown 打开(WsPopover)
[2] 选「📄 空白工作流」→ emit('select', { kind: 'blank' })
[3] WorkflowView.handleCreate() 读 useEditorPreferences.addMethods(确认)
[4] await useWorkflowClient.mockCreate(WorkflowFactory.blank())
    └─ dev-mock-store.mockCreate:写 localStorage + 返回新定义
[5] dispatchEvent('worldsmith:workflow-list', { action: 'create', definition })
    └─ useWorkflow.handleWorkflowList 收到 → refreshList() 更新 workflowList
[6] workflowList.value.push(newSummary) → WorkflowList 重渲染
[7] currentWorkflowId = newId; currentTab = 'editor'
[8] WorkflowEditorView 加载新工作流,ThreeViewLayout 挂载
[9] 画布自动放置 Start 节点(画布中心),光标定位
[10] useWorkflowPersistence 自动 debounce 300ms 后保存草稿

▌ 场景 B:从模板创建

[1-2] 同上,但 select.kind = 'from-template'
[3] WorkflowView 打开 WsModal(TemplateLibrary)
[4] 用户点模板卡 → emit('pick', template)
[5] WorkflowView.handleCreateFromTemplate(template)
    └─ WorkflowFactory.fromTemplate(template):展开 nodes/edges
[6] await mockCreate(definition) → [同 4-9]
[7] 关闭 WsModal;切到 editor tab;画布 fitView() 让所有节点可见

▌ 场景 C:agent_decision 暂停 → 决策 → 继续

[1] RunView 订阅 'worldsmith:workflow-run' 事件
[2] 运行到 agent_decision 节点 → 后端 dispatch 'workflow_node_paused'
    └─ useWorkflowRuns 收到 → 打开 AgentDecisionCard(非模态)
[3] AgentDecisionCard 显示 context / question / options(从节点配置读)
[4] 启动计时器(默认 5min timeout)
[5] 用户点击某个 option → emit('decide', { choice, note? })
[6] RunView.handleDecide():await workflow_continue({ runId, decision: choice })
    └─ Tauri / dev-mock:写入决策;继续运行
[7] AgentDecisionCard 关闭;画布节点从 'awaiting-decision' → 'running' → 'completed'
[8] 决策事件写入 run.detail.log(WsCodeBlock 可查看)
[9] 若 5min 内无操作 → 自动 fallback 到节点配置 defaultOption;emit 同样事件

边界:若用户在 5min 内关掉卡片,后台计时继续;再次进入仍可见剩余时间

5.6 风险与对策

  • R1:WsButton 加 gradient variant 后跟现有 primary 撞色
    对策:gradient 仅用于"主 CTA"场景,其他处仍用 primary;色板用 --gradient-accent 已有 token,不是新色。
  • R2:多范式编辑模式同时挂载会显著拖慢首屏
    对策:按需 defineAsyncComponent,首次只挂载当前 editMethod 对应的组件;切换偏好后再 lazy load 另一个。画布初次进入 < 200ms 增量。
  • R3:HTML5 drag 在某些 webview 行为不一致(Tauri webkitgtk 已知坑)
    对策:同时实现 click 添加作为 fallback(已默认开启);drag 仅增强;拖拽失败的兜底是"点击加入 + 弹窗选位置"。
  • R4:agent_decision 5min 超时可能在某些场景过短 / 过长
    对策:超时时长纳入节点配置(per-node override),默认 5min;设 0 = 永不超时(等用户决策)。
  • R5:新原语发到 src/ui/ 后被 Agent 工具页 / 设置页等无关位置误用
    对策:文件头加 @internal workflow-plugin JSDoc 注释;不导出 index.ts,仅本插件可显式 import。

方案 A 智能映射：JSON → A2UI 组件

根据工具名称和 JSON 结构，自动选择最合适的 A2UI 组件渲染。entity_list → EntityCard 列表，entity_create → 成功卡片，daily_report → 统计面板，error → 错误卡片。

方案 B 通用结构化：JSON → 键值对卡片

不区分工具类型，统一将 JSON 解析为键值对列表。成功/错误用颜色区分，嵌套对象递归展开。通用但信息密度较低。

  • 实体列表没有卡片感
  • 信息密度不如方案 A
方案 C 混合方案：已知工具智能映射 + 未知工具通用兜底

对已知工具（entity_list, entity_create, daily_report 等）使用方案 A 的专属 UI，对未知工具自动降级为方案 B 的键值对卡片。两全其美。
